### 主要收获

#### 黑客松项目整体架构确定

项目定位：**链上信号机器人 + 半自动交易执行系统**，三个核心功能模块：

**模块一：资金费率监控**
- 数据源：币安/OKX 公开 WebSocket API，无需 API Key，免费稳定
- 检测逻辑：永续合约资金费率突然超过 ±1%/8h，即推送飞书信号
- 套利逻辑：费率为正则做空+做多（收红包），费率为负则反向
- 注意：币安永续是 CEX，Hermes 无法直接帮开仓；需桥接链上 DEX（GMX、dYdX）

**模块二：低波动币种检测 + 刷量农夫**
- 检测方法：计算 N 小时内价格标准差，识别 annualized volatility < 0.5% 的币种
- 横盘信号：价格横盘通常意味着有人在吸筹或刷量，酝酿大波动
- 执行策略：低波动 → 在 dYdX/GMX/Vertex 刷小额 swap 循环（0.01 ETH 每次，10次循环）
- 目的：刷交易量换取空投积分（dYdX 交易量排名、GMX 分润、Vertex 积分）

**模块三：聪明钱地址打分系统**
- 核心指标：筛选"中等速度 + 稳定逻辑"的钱包，单次收益率 ≥ 50% 作为硬门槛
- 三维评分体系：
  - 收益率分（40%）：单次 ROI ≥ 50% 的交易次数 / 总交易次数
  - 稳定性分（30%）：夏普比率估算连续盈利概率
  - 速度分（30%）：平均持仓时间（不是最快，足够快即可）
- "吃鱼只吃中间段"策略：监控聪明钱地址的链上 ERC-20 Transfer 事件，跟随买入信号，聪明钱离场后我们也平仓
- 辅助判断：Gas 费模式（判断是否机器人化）、链上签名频率

### 遇到的问题
- 币安永续是 CEX，无法直接用智能合约操作，需要找链上替代品（GMX、dYdX）
- 聪明钱打分需要历史链上数据，数据源选型待定（DeBank / Nansen / Dune）

### 解决方案
- CEX 永续套利仅作为信号通知，用户手动操作；链上 DEX 套利可自动执行
- 链上数据：先用 The Graph + 自己扫事件做 MVP，Hackathon 期间不依赖付费 API
